# 헤어 + 메이크업 다중 스텝 예약 기능 가이드

## 📋 개요

고객이 "헤어 + 메이크업" 서비스를 선택하면 헤어 담당자와 메이크업 담당자를 각각 선택할 수 있으며, 관리자 페이지의 타임라인에 순서대로 두 스케줄이 표시됩니다.

## 🔧 구현 내용

### 1. 데이터베이스 변경

**추가된 컬럼:**
- `bookings.staff_sequence` (JSONB 타입)
  - 형식: `[{"staffId": "uuid-1", "dur": 40}, {"staffId": "uuid-2", "dur": 60}]`
  - 예시: 헤어(40분) → 메이크업(60분) 순서

**마이그레이션 실행:**
```sql
-- Supabase SQL Editor에서 실행
ALTER TABLE bookings
ADD COLUMN IF NOT EXISTS staff_sequence JSONB;
```

또는 `migration_add_staff_sequence.sql` 파일을 Supabase SQL Editor에서 실행하세요.

### 2. 고객 예약 페이지 (customer.html)

**변경 사항:**
- "헤어 + 메이크업" 선택 시 두 개의 스텝 선택 UI 표시
  - 헤어 담당 디자이너 선택
  - 메이크업 담당 디자이너 선택
- 예약 가능 시간 체크 로직이 두 스텝의 스케줄을 모두 고려
- 예약 제출 시 `staff_sequence` 데이터 생성

### 3. 관리자 페이지 (admin.html)

**기존 기능 활용:**
- `bookingToBlock` 함수가 이미 `staff_sequence`를 처리
- 여러 스텝이 있는 예약은 각 스텝별로 개별 블록 생성
- 타임라인에 순서대로 표시

## 🧪 테스트 방법

### 1. 데이터베이스 마이그레이션

1. Supabase 대시보드 접속
2. SQL Editor 열기
3. `migration_add_staff_sequence.sql` 파일 내용 복사 & 실행
4. "✅ staff_sequence 컬럼 추가 완료!" 메시지 확인

### 2. 고객 예약 테스트

**시나리오 1: 헤어 + 메이크업 예약**

1. 고객 예약 페이지 접속 (`customer.html` 또는 배포된 URL)
2. "✨ 헤어 + 메이크업" 서비스 선택
3. 헤어 담당 디자이너 선택 (예: 보니)
4. 메이크업 담당 디자이너 선택 (예: 지현)
5. 날짜 선택
6. 시간 선택 (예: 10:00)
   - 시스템이 자동으로 두 스텝의 스케줄을 확인
   - 헤어 (10:00~10:40) + 메이크업 (10:40~11:40)
7. 고객 정보 입력
   - 이름: 테스트 고객
   - 연락처: 010-1234-5678
8. "예약 요청하기" 클릭
9. 성공 메시지 확인

**시나리오 2: 단일 서비스 예약 (기존 기능)**

1. "💇‍♀️ 헤어 스타일링" 또는 "💄 메이크업" 선택
2. 담당 디자이너 한 명만 선택
3. 날짜 및 시간 선택
4. 고객 정보 입력 후 예약

### 3. 관리자 페이지 확인

1. 관리자 페이지 접속 (`admin.html` 또는 배포된 URL)
2. 비밀번호 입력: `nunu1122!`
3. 타임라인에서 예약 확인
   - **헤어 + 메이크업 예약:** 두 개의 블록이 순서대로 표시
     - 첫 번째 블록: 헤어 담당자의 타임라인에 표시
       - 예: 10:00~10:40 (보니)
       - 라벨: "✨ 헤어+메이크업 [요청중] (1/2)"
     - 두 번째 블록: 메이크업 담당자의 타임라인에 표시
       - 예: 10:40~11:40 (지현)
       - 라벨: "✨ 헤어+메이크업 [요청중] (2/2)"
   - **단일 서비스 예약:** 한 개의 블록만 표시

### 4. 예약 충돌 테스트

1. 첫 번째 예약: 헤어(보니) + 메이크업(지현) - 10:00 시작
   - 보니: 10:00~10:40
   - 지현: 10:40~11:40
2. 두 번째 예약 시도: 같은 날짜, 다른 시간
   - **10:30 선택 시도:** 보니가 이미 예약되어 있어 비활성화
   - **10:40 선택 시도:** 지현이 이미 예약되어 있어 비활성화
   - **11:40 선택:** 두 스텝 모두 가능하므로 활성화

## 🔍 디버깅

### 예약이 생성되지 않을 때

1. 브라우저 개발자 도구 (F12) 열기
2. Console 탭에서 에러 확인
3. Network 탭에서 Supabase API 요청 확인

### 타임라인에 표시되지 않을 때

1. 관리자 페이지에서 F12 열기
2. Console에서 `bookingToBlock` 함수 출력 확인
3. `staff_sequence` 파싱 오류 여부 확인

### 시간 충돌 확인이 작동하지 않을 때

1. `availableTimes` 계산 로직 확인
2. `bookings` 데이터가 올바르게 로드되었는지 확인
3. `staff_id` 매칭이 올바른지 확인

## 📝 참고사항

- 기존 예약 데이터는 영향받지 않음 (`staff_sequence`가 NULL인 경우 기존 로직 사용)
- `requested_staff_id`는 첫 번째 스텝(헤어)을 대표로 저장
- 관리자 페이지의 `bookingToBlock` 함수가 자동으로 단일/다중 스텝을 처리
- 예약 시간 계산: 시작 시간 + 헤어 소요시간 + 메이크업 소요시간 = 종료 시간
