# 고객 검색 기능 추가 완료

## ✅ 구현 내용

### 📱 검색창 추가
미니 달력 상단에 고객 검색창이 추가되었습니다.

**위치**: 스케줄 페이지 → 날짜 클릭 → 미니 달력 최상단

### 🔍 검색 기능

#### 1. 검색 방법
- **입력**: 고객명 또는 연락처 입력
- **실행**: 
  - 🔍 버튼 클릭
  - 또는 Enter 키 입력
- **초기화**: X 버튼 클릭

#### 2. 검색 결과 표시
- **캘린더에 표시**:
  - 검색된 예약이 있는 날짜: 노란색 배경 + 주황색 테두리
  - 빨간 점: 해당 날짜의 예약 개수 (최대 5개)
  
- **검색 결과 안내**:
  - 상단에 주황색 배너 표시
  - 예: "📌 '김철수' 검색 결과: 3건 (2일)"

- **예약 리스트**:
  - 선택된 날짜의 검색된 예약만 표시
  - 시간순 정렬

#### 3. 검색 예시
```
입력: "김철수" → 김철수님의 모든 예약 표시
입력: "010-1234" → 010-1234로 시작하는 연락처의 모든 예약 표시
입력: "1234" → 연락처에 1234가 포함된 모든 예약 표시
```

### 🎨 UI/UX

#### 검색창 디자인
- 크기: 36px 높이

- X 버튼: 검색어가 입력되면 자동 표시
- 🔍 버튼: 검색어가 있을 때 활성화

#### 캘린더 시각적 표시
- **검색 결과 날짜**:
  - 배경: 노란색 (#FEF3C7)
  - 테두리: 주황색 (2px solid #F59E0B)
  - 텍스트: 진한 갈색 (#92400E)
  - 점: 주황색 (#F59E0B)

- **일반 예약 날짜**:
  - 점: 빨간색 (#EF4444)

#### 도움말
- 검색 모드: "노란색 날짜: 검색된 예약 · 날짜를 다시 클릭하면 타임라인으로 이동합니다"
- 일반 모드: "날짜를 다시 클릭하면 타임라인으로 이동합니다"

### 🔄 동작 흐름

1. **검색 실행**:
   - 고객명/연락처 입력
   - 🔍 버튼 클릭 또는 Enter
   - 검색 모드 활성화

2. **결과 표시**:
   - 검색 결과 안내 배너 표시
   - 해당 고객의 예약이 있는 날짜 노란색으로 강조
   - 검색 결과의 첫 번째 날짜로 자동 이동
   - 캘린더가 해당 월로 이동

3. **날짜 선택**:
   - 노란색 날짜 클릭
   - 하단 리스트에 해당 날짜의 예약 표시

4. **검색 초기화**:
   - X 버튼 클릭
   - 검색 모드 해제
   - 전체 예약으로 복귀

### 📊 기술 구현

**검색 로직**:
```javascript
// 대소문자 구분 없음
// 하이픈(-) 자동 제거하여 비교
const q = searchQuery.toLowerCase().replace(/-/g,'');
const name = b.customers?.name?.toLowerCase() || '';
const phone = b.customers?.phone?.replace(/-/g,'') || '';
return name.includes(searchQuery) || phone.includes(q);
```

**상태 관리**:
- `searchQuery`: 검색어
- `searchMode`: 검색 모드 활성화 여부
- `searchResults`: 검색 결과 (확정 예약만)
- `searchDates`: 검색 결과의 날짜 목록

**성능 최적화**:
- `useMemo`로 검색 결과 캐싱
- 검색 모드일 때만 검색 필터 적용
