# 버그 수정 - 2026년 6월 7일

## 🐛 문제 상황

미니 달력 클릭 시 에러가 발생하는 문제

### 증상
- 미니 달력 열기 시 에러 발생
- 자동완성 기능이 정상 작동하지 않음
- 특정 상황에서 undefined 참조 에러

## 🔍 원인 분석

### 1. Optional Chaining 문제
```javascript
// 문제 코드
onClose?.();
```
- `onClose`가 undefined일 때 호출 자체는 안전하지만, 코드 흐름에서 예기치 않은 동작 발생

### 2. Props 누락
```javascript
// S1Date에서 MiniCalendar 호출 시
<MiniCalendar value={draft.date} onChange={d=>set({...draft,date:d})}/>
// supaCustomers, onCreateBooking props 없음
```
- `supaCustomers`가 undefined인 상태에서 자동완성 로직 실행 시도
- 에러는 발생하지 않지만 불필요한 연산 수행

### 3. 상태 업데이트 누락
```javascript
// onCreateBooking 콜백에서
setFlow('s3');
// setShowCal(false) 누락으로 미니 달력이 열린 채로 유지됨
```

## ✅ 해결 방법

### 1. onClose 호출 안정화
```javascript
// Before
if(onCreateBooking){
  onCreateBooking(customer,selectedDate);
  onClose?.();
}

// After
if(onCreateBooking){
  onCreateBooking(customer,selectedDate);
}
if(onClose){
  onClose();
}
```
**효과**: 명시적인 조건 체크로 코드 의도 명확화

### 2. 자동완성 기능 가드 추가
```javascript
// 자동완성 사용 가능 여부 체크
const hasAutocompleteFeature=supaCustomers&&supaCustomers.length>0&&onCreateBooking;

// 자동완성 표시 조건
onFocus={()=>searchQuery.trim()&&hasAutocompleteFeature&&setShowAutocomplete(true)}

// 드롭다운 렌더링 조건
{hasAutocompleteFeature&&showAutocomplete&&autocompleteCustomers.length>0&&...}
```
**효과**: 필요한 props가 모두 있을 때만 자동완성 기능 활성화

### 3. 미니 달력 닫기 추가
```javascript
// Before
setDraft(d);
setEditingId(null);
setFlow('s3');

// After
setDraft(d);
setEditingId(null);
setShowCal(false); // 미니 달력 닫기
setFlow('s3');
```
**효과**: 예약 생성 플로우 시작 시 미니 달력 자동 닫힘
